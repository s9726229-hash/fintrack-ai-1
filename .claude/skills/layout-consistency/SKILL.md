# FinTrackAI 排版調整（全域，桌面/手機共用）

## 工作流程

通用的合作方式（先找根因、每次都要提方案確認、跨頁調整按階層橫向做完再往下層、用真實資料驗證、驗證OK立刻補文件、美學決策交給設計類skill）記在全域的 `iterative-layout-workflow` skill（`~/.claude/skills/iterative-layout-workflow/SKILL.md`），這裡不重複，照那份執行。這份文件只留 FinTrackAI 專屬的具體規則。

## 版面間距規則

- **Padding 只有一層**：`components/Layout.tsx` 的內容容器統一管邊距（手機 `px-3 pt-3 pb-24`，桌面 `md:p-8 md:pb-8`）。各 `views/*.tsx` 根 `<div>` 只加 `md:p-6`，不要再加手機版 padding 或 `pb-24`。
- **標籤＋數值不要同行擠壓**：短標籤＋可能很長的數值，改成標籤在上、數值在下兩行，不要橫向搶位置導致標籤被截斷。**數值本身過大時要簡化顯示**（例如 1,350,000 縮寫成 1.35M 或 135萬），不要單純縮小字級硬塞——縮寫後版面才不會被迫加寬或造成換行，`services/format.ts` 的 `formatMoneyCompact` 已有這個邏輯可直接重用。
- **金額類小卡防止換行**：數值 `<div>` 加 `whitespace-nowrap`；3 欄並排等窄版面用 `text-[15px]` 而非 `text-base`；必要時卡片 padding 收緊到 `p-2.5`。
- **輔助資訊獨立成行**：日期區間、統計摘要這類輔助文字不要跟主要操作列（tabs、按鈕）擠同一個 `flex-wrap` row，各自獨立成行。
- **`CollapsibleSection`（Dashboard.tsx）要有手機斷點**：標題列 `px-4 py-3 md:px-6 md:py-4`，展開內容 `px-4 pb-4 md:px-6 md:pb-6`，不要用單一固定值。
- **行內表單輸入框用 `flex-1`**，不要寫死窄寬度（如 `w-16`）。
- **Mobile Header 邊距要跟內容對齊**：`px-4` / `h-14`，跟內容區風格一致。

## 頁面標準結構（6 層，由上到下）

1. **全域頂部列**：左＝App 名稱，右＝全域功能。對應 `MobileHeader.tsx` / `Sidebar.tsx`，不用每頁重做。
2. **頁面標題列**：左＝頁名＋icon＋副標題，右＝該頁全域功能鍵（無則留空）。主要操作按鈕一律用 `components/ui.tsx` 的共用 `Button` 元件，不寫原生 `<button>`。
3. **分頁列**：左＝分頁選項，右＝分頁專屬功能鍵。**分頁定義＝任何「切換後會改變下方資料內容」的按鈕組**（換整組資料集、篩選子集、選時間範圍都算），一律用共用 `Tabs` 元件（`components/ui.tsx`，底線樣式），放在標題列正下方。頁面本身已有分頁時，巢狀的次級分頁就放在該分頁內容最頂端。
4. **KPI／統計小卡**：標題壓縮扁平化，留白比例約 20%。數字防換行見上方規則。項目多（5+）時用長方形整合卡，不要硬塞更多小方塊。分析型／洞察卡（健康診斷、財務摘要類）算這層延伸。**只有單一一張小卡時，靠左對齊，不要撐滿整行寬度**（例如 Budget 的「本月總預算執行率」目前是撐滿整行的大卡，之後要調整成靠左）。
5. **篩選功能**：緊接 KPI 卡之後、列表之前，是「已選定分頁範圍內再細篩」的功能（搜尋框、日期區間顯示）。跟第3層分頁不同層級，不要混在一起。
6. **資料列表**：放最下面，捲動時欄位表頭要 sticky。**不分桌面/手機，只用一套統一標記**（不要再分 `<table>` 桌面版＋手機卡片版兩套），以 `TransactionList.tsx` 的單一 div-list 結構為範本：sticky 標題列（名稱＋筆數）＋ sticky 欄位標籤列＋資料列。**字體固定一套尺寸，不用 `md:` 斷點切換**：名稱／金額 14px 粗體（`text-sm font-bold`）、次要說明文字 11px（`text-[11px]`）、圖案徽章與操作按鈕也固定一種大小。

   **單筆資料列固定「圖案／名稱／數據／操作」四段**：
   - **圖案**：一律用小圓點或帶 icon 的圓形徽章（不用實心色條，色條在窄版面/長列表堆疊時視覺過重），不可省略。
   - **名稱**：主體名稱 + 次要說明合併成一行小字（例如「類別＋日期」「股數×成交價＋日期」），`flex-1 min-w-0`。桌面版原本獨立的欄位（最後更新日期、現價/均價、成交價、日期）都折進這行次要說明或直接不顯示，不再開獨立欄位——資料列表統一之後，欄位數就是這一套，不因裝置而增減。
   - **數據**：金額/數值，`shrink-0 whitespace-nowrap`，需要的次要數據（漲跌%、損益）另起一行。
   - **操作**：編輯/刪除/更新等按鈕，最後、`shrink-0`。

   **目前三個列表統一後的欄位**（2026-07-22 確認）：
   | 列表 | 圖案 | 次要說明 | 數據 |
   |---|---|---|---|
   | 資產管理 | 類別小圓點/icon 徽章 | 類別＋最後更新日期（負債另加還款進度條） | 金額（含投資損益徽章） |
   | 股票庫存 | 損益色點徽章 | 股票代號＋持股股數 | 市值＋損益%（現價/均價不再顯示於列表，看「更新現價」或編輯） |
   | 交易紀錄 | 買/賣徽章 | 股數×成交價＋日期 | 總金額＋損益/報酬率 |

### 已知例外
- **Dashboard 暫緩套用分頁/結構統一化**（使用者 2026-07-21 指示先不動），內部圖表切換與債務卡分頁未來仍要換成共用 `Tabs`。
- **Recurring 資料列表維持卡片格狀**，不強改 sticky 列表（項目少，格狀更合適）。
- **Budget/Guide/Settings 標題列右側留空**是正確的（無對應全域功能）。

## 驗證方式

1. `preview_start` 啟動 `FinTrack AI Dev Server`（port 3000），`resize_window` 切到 `mobile`（375x812）。
2. **用真實備份資料**（去敏後）灌進 localStorage 測試，不要只看空狀態——排版問題常只有真實資料量才會暴露。去敏與灌入方法見 [[project-fintrackai-dividend-verification]]。
3. `read_page` + `computer` 切換分頁，`screenshot` / `get_page_text` 確認結果。
4. 改完也用 `1024x768` 桌面尺寸確認沒被波及（只動 mobile-only class 時）。
5. **工具怪癖**：桌面寬螢幕（如 1280x900）截圖偶爾會顯示內容擠壓在左上角，且同時 `computer` 點擊可能失效——這是顯示/互動工具本身的問題，不是排版壞了。用 `javascript_tool` 確認 `window.innerWidth`／`getBoundingClientRect()` 判斷真實狀態；點擊失效時改用 `document.elementFromPoint(x,y).closest('button').click()` 直接觸發。
